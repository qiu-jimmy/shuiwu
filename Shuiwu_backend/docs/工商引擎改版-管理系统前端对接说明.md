# 工商引擎改版 · 管理系统前端对接说明

> 文档用途：告知管理系统前端，本次工商引擎改版中后端新增的接口字段、枚举值变化及详情页展示规范，无需其他文档。

---

## 变更记录

| 版本 | 日期 | 说明 |
|------|------|------|
| v1.0 | 2026-03-03 | 初版：新增 `license_application`（工商执照申请）申报类型，相关字段与展示规范 |

---

## 一、变更概述

| 项目 | 变更前 | 变更后 |
|------|--------|--------|
| 申报类型枚举 | 5 种（年报/变更/注销/税务/发票） | 新增第 6 种：**`license_application`（工商执照申请）** |
| 统计接口返回 | 无 `license_application_count` | 新增 `license_application_count` 字段 |
| 详情接口 | 所有类型字段结构相同 | `license_application` 类型详情有独立字段在 `declaration_info` 中 |
| 类型筛选 | 不含 `license_application` | 筛选列表新增 `license_application` |

---

## 二、申报类型枚举变化

### 接口：`GET /api/business-declaration/meta/types`

新增返回一条：

```json
{ "code": "license_application", "name": "工商执照申请" }
```

**完整枚举（更新后）：**

| code | 中文名 |
|------|--------|
| `annual_report` | 年报 |
| `change_registration` | 变更登记 |
| `deregistration` | 注销登记 |
| `tax_registration` | 税务登记 |
| `invoice_application` | 发票申请 |
| `license_application` | **工商执照申请（新增）** |

> **前端需要做的：** 所有申报类型下拉筛选框、类型标签展示，新增 `license_application → 工商执照申请` 映射。

---

## 三、列表接口变化

### 接口：`GET /api/business-declaration/admin/list`

**Query 参数 `declaration_type` 新增可选值：**

```
declaration_type=license_application
```

列表返回数据结构**不变**，`declaration_type` 字段直接展示新值，前端只需补充中文名映射：

```js
// 类型展示映射（新增最后一条）
const TYPE_LABEL = {
  annual_report:        '年报',
  change_registration:  '变更登记',
  deregistration:       '注销登记',
  tax_registration:     '税务登记',
  invoice_application:  '发票申请',
  license_application:  '工商执照申请',  // 新增
}
```

## 五、详情接口变化

### 接口：`GET /api/business-declaration/{declaration_id}`

**整体结构不变**，对 `declaration_type = license_application` 的申报，各字段含义如下：

### 5.1 顶层字段映射关系

| 字段名 | 含义（执照申请语境） | 备注 |
|--------|----------------------|------|
| `business_name` | 主营业执照字号（即前端 `license_store_name`） | 映射存储 |
| `operator_name` | 申请人姓名（即前端 `applicant_name`） | 映射存储 |
| `operator_phone` | 申请人手机号（即前端 `applicant_phone`） | 映射存储 |
| `operator_id_card` | 申请人身份证号（即前端 `id_card_number`） | 映射存储 |
| `attachments` | 补充材料列表（即前端 `extra_attachments`） | 结构：`[{file_url, name}]` |
| `declaration_info` | 执照申请专属信息（见下方 5.2） | JSONB 对象 |

### 5.2 `declaration_info` 字段结构（`license_application` 专属）

```json
{
  "license_store_name_list": ["备用字号一", "备用字号二"],
  "id_card_valid_type": "range",
  "id_card_valid_start": "2010-06-01",
  "id_card_valid_end": "2030-06-01",
  "id_card_front_url": "https://oss.../id_front.jpg",
  "id_card_back_url": "https://oss.../id_back.jpg",
  "political_status": "masses",
  "education_level": "bachelor",
  "email": "zhangsan@example.com",
  "agree_protocol": true
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `license_store_name_list` | string[] | 备用字号列表，可为空数组 |
| `id_card_valid_type` | string | `range`（有期限）/ `long_term`（长期） |
| `id_card_valid_start` | string | 证件有效期开始，`YYYY-MM-DD`，`long_term` 时为 `null` |
| `id_card_valid_end` | string | 证件有效期结束，`YYYY-MM-DD`，`long_term` 时为 `"2099-12-31"` |
| `id_card_front_url` | string | 身份证人像面图片 OSS URL |
| `id_card_back_url` | string | 身份证国徽面图片 OSS URL |
| `political_status` | string | 见下方枚举 |
| `education_level` | string | 见下方枚举 |
| `email` | string | 邮箱，可为空字符串 |
| `agree_protocol` | boolean | 固定为 `true` |

### 5.3 `political_status` 枚举

| 后端值 | 中文展示 |
|--------|----------|
| `masses` | 群众 |
| `league_member` | 共青团员 |
| `party_member` | 中共党员 |
| `other` | 其他 |

### 5.4 `education_level` 枚举

| 后端值 | 中文展示 |
|--------|----------|
| `high_school` | 高中及以下 |
| `junior_college` | 大专 |
| `bachelor` | 本科 |
| `postgraduate` | 研究生及以上 |

## 六、管理系统详情页展示建议

当 `declaration_type === 'license_application'` 时，建议在详情页展示以下分区：

### 卡片一：基本信息
| 展示标签 | 数据来源 |
|----------|----------|
| 申报单号 | `declaration_no` |
| 申报类型 | `declaration_type` → "工商执照申请" |
| 主字号 | `business_name` |
| 备用字号 | `declaration_info.license_store_name_list`，逗号分隔展示 |
| 提交时间 | `created_at` |
| 当前状态 | `status` |

### 卡片二：申请人信息
| 展示标签 | 数据来源 |
|----------|----------|
| 姓名 | `operator_name` |
| 手机号 | `operator_phone` |
| 身份证号 | `operator_id_card` |
| 证件有效期类型 | `declaration_info.id_card_valid_type` → "有期限"/"长期" |
| 证件有效期 | `id_card_valid_type=range` 时展示 `start ~ end`；`long_term` 时展示"长期" |
| 政治面貌 | `declaration_info.political_status`（参考 5.3 枚举映射） |
| 学历 | `declaration_info.education_level`（参考 5.4 枚举映射） |
| 邮箱 | `declaration_info.email`，空字符串则展示"-" |

### 卡片三：证件图片
| 展示标签 | 数据来源 |
|----------|----------|
| 身份证（人像面） | `declaration_info.id_card_front_url`，图片预览 |
| 身份证（国徽面） | `declaration_info.id_card_back_url`，图片预览 |

### 卡片四：补充材料
- 数据来源：`attachments`（数组）
- 结构：`[{ "file_url": "...", "name": "..." }]`
- 展示：文件名 + 可点击下载/预览链接；`attachments` 为 `null` 或空数组时展示"暂无补充材料"

### 卡片五：用户备注
- 数据来源：`user_remarks`，为空时展示"-"

## 七、未变动项

- 处理申报接口 `POST /api/business-declaration/admin/{id}/process`：**无变化**，执照申请与其他类型使用相同的处理流程和参数
- 状态枚举（`pending/processing/completed/rejected/need_supplement`）：**无变化**
- Token 鉴权方式：**无变化**
- 列表接口分页结构：**无变化**
- 用户端申报历史列表 `/api/business-declaration/list`：**无变化**（同样支持 `declaration_type=license_application` 筛选）
