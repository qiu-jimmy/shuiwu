# 智能税务系统接口分析报告（简化版）

## 一、系统概述

本系统基于**亿企赢开放平台**提供的45个API接口，实现智能发票、智能报税两大核心功能。

---

## 三、核心业务流程

### 3.1 开票流程
```
查额度(25) → 提交开票(11) → 查询结果(12) → 下载发票(47)
```

### 3.2 报税流程
```
获取发票(1+2) → 预填申报(18/21/22) → 提交税局 → 下载完税证明(26)
```

### 3.3 进项签收流程
```
获取进项(1) → 发票查验(9+10) → 批量签收(3)
```

## 四、关键技术点

### 4.1 统一认证
所有接口需要：
- `appKey`：应用密钥
- `timestamp`：Unix时间戳
- `signature`：签名结果串
- `Content-type`：application/json 或 multipart/form-data

### 4.2 税种代码
| 代码 | 税种 |
|------|------|
| 10101 | 增值税-一般纳税人 |
| 10102/10103 | 增值税-小规模（月/季报） |
| 10320 | 消费税 |
| 21175-21179 | 印花税（按次/月/季/半年/年） |
| 29890-29892 | 财务报表（月/季/年报） |

## 六、重要说明

###  接口支持的功能
- 发票开具、查验、签收、管理
- 税务申报表预填（数据准备）
- 财务报表预填
- 完税证明下载
- 基础记账功能

###  接口限制
1. **预填≠申报**：接口18/21/22等是预填接口，只负责填充数据，实际提交税局需要额外对接
2. **无工商接口**：当前接口不包含工商注册、变更、年报的直接接口
3. **异步任务**：部分接口（如开票、查验）需要轮询或等待异步通知

## 七、总结

本系统通过45个接口，完整覆盖：
-  智能发票（开票、查验、签收）
-  智能报税（增值税、消费税、印花税、财务报表）
-  基础记账（账套、科目、凭证）

适用于企业日常税务管理场景。

## 八、核心功能详解

### 8.1 智能开票

####  能做什么
- 数电专票/普票开具（支持蓝字、红字发票）
- 建筑服务、货物运输、不动产销售等特殊业务发票
- 发票额度实时查询
- 发票状态跟踪（开票中→成功→交付）
- 发票PDF/XML/OFD文件下载
- 发票作废、红冲

####  怎么做
```
技术实现流程：
1. 调用接口25查询可用发票额度
2. 填写购销双方信息、商品明细
3. 调用接口11提交开票申请
4. 轮询接口12查询开票状态（每5秒）
5. 开票成功后调用接口47下载发票文件
```

**使用接口**：接口25、11、12、47、15

### 8.2 智能报税

####  能做什么
- 增值税申报（一般纳税人、小规模纳税人）
- 消费税、印花税申报
- 财务报表填报（资产负债表、利润表等）
- 申报结果查询展示
- 完税证明下载
- 进销项发票数据汇总

####  怎么做
```
技术实现流程：
1. 调用接口1、2获取进销项发票数据
2. 根据纳税人类型选择预填接口：
   • 一般纳税人 → 接口18
   • 小规模纳税人 → 接口21
3. 调用接口22预填财务报表
4. 展示预填数据供客户确认
5. 客户确认后通过亿企代账提交税局
6. 调用接口26下载完税证明
```

**使用接口**：接口1、2、18/21、22、26
