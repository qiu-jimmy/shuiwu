# 智能税务系统接口分析报告

## 一、系统概述

本系统基于第三方税务平台（亿企赢）提供的开放API，实现智能报税、智能发票、工商申报等核心税务功能。系统共有 **47个接口**，分为8大功能模块。

---

## 二、接口分类与功能说明

### 2.1 智能发票模块（核心）

#### 2.1.1 开票接口

| 接口ID | 接口名称 | 接口路径 | 功能说明 |
|--------|----------|----------|----------|
| **11** | 开票信息提交 | `/gateway/stpopenapi/invoiceBill/save` | 提交开票申请，支持数电专票/普票，支持蓝字发票开具、红字发票冲红、特殊业务发票（建筑服务、货物运输、不动产销售/租赁、旅客运输等） |
| **12** | 开票结果查询 | `/gateway/stpopenapi/invoiceBill/query` | 查询开票状态、发票详情、PDF/XML/OFD文件下载地址，支持按单据编号、时间范围查询 |
| **13** | 开票结果异步通知 | - | 接收开票完成的异步回调通知 |
| **15** | 发票作废红冲 | `/gateway/stpopenapi/invoiceBill/red` | 对已开票的单据进行红冲或作废操作 |
| **25** | 发票额度查询 | `/gateway/stpopenapi/invoice/common/queryBillingDataOverview` | 查询企业可用发票额度（总授信额度、剩余额度、本月可领用票数） |
| **47** | 获取发票文件 | `/gateway/stpopenapi/invoiceBill/getInvoiceFile` | 获取发票的PDF、XML、OFD格式文件 |

#### 2.1.2 进销项发票管理

| 接口ID | 接口名称 | 接口路径 | 功能说明 |
|--------|----------|----------|----------|
| **1** | 进项发票获取（批量） | `/gateway/stpopenapi/invoice/getInputInvoicePageList` | 批量获取进项发票列表，支持按税款所属期、开票日期、修改日期筛选，可获取商品明细 |

| **5** | 发票获取任务结果查询 | `/gateway/stpopenapi/invoice/fetchTask/query` | 查询异步任务的获取结果 |

| **45** | 修改签收业务信息（批量） | `/gateway/stpopenapi/invoice/batch/modifySign` | 修改已签收发票的抵扣状态、会计备注等业务信息 |

### 6.2 关键参数说明

- `timestamp`: Unix时间戳
- `version`: 接口版本（默认1.0.0）
- `appKey`: 平台颁发的访问密钥
- `signature`: 签名结果串（需按规则计算）
- `Content-type`: application/json 或 multipart/form-data

**通用参数：**
- `taxNumber` / `taxNo` / `nsrsbh`: 纳税人识别号（企业税号）
- `phone`: 亿企赢平台登录手机号（部分接口需要）

## 七、技术实现建议

### 7.1 认证与签名

所有接口需要实现统一的签名机制：
1. 构建请求参数JSON字符串
2. 按照平台提供的签名算法生成签名
3. 在请求头中携带 `appKey`、`timestamp`、`signature`

### 7.2 异步任务处理

对于大数据量场景，建议使用异步任务接口：
- 发票获取：接口4 + 接口5
- 查验任务：接口1705240565347459100 + 接口1705240565347459101
- 数电原件：接口6 + 接口7

### 7.3 轮询机制

对于需要等待结果的接口（如开票、查验），建议：
- 前端：每5秒轮询一次，最多轮询20次（100秒）
- 后端：使用消息队列处理异步通知（接口13）

## 八、总结

本系统通过47个接口，完整覆盖了智能报税、智能发票、工商申报（客商风控）三大核心业务场景。系统设计清晰，接口职责明确，可以满足企业日常税务管理需求。

### 核心优势：
1. **全流程覆盖**：从发票开具、查验、签收到报税申报、完税证明下载
2. **智能化程度高**：支持OCR识别、预填申报、自动计算
3. **数据完整性**：进销项发票管理、财务报表、记账凭证一体化
4. **风控能力**：客商风险查询、发票查验保障交易安全

### 缺失功能：
1. **工商注册/变更**：当前接口未包含工商注册、变更、年报的直接接口
2. **税务申报提交**：预填接口（18/21/22等）只是预填写数据，实际提交税局需要额外对接
